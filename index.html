Programación Orientada a Objetos en Java

Aquí podrás aprender desde lo más básico como qué es y para qué sirve la OOP hasta las prácticas que un buen programador avanzado en Java utilizaría.

1

Introducción a la POO y entorno de desarrollo en Java

IntelliJ IDEA, Eclipse, NetBeans, VS Code, JDK, JVM y paquetes

Explicación conceptual

La Programación Orientada a Objetos (POO) es un paradigma que organiza el código en "objetos", los cuales contienen datos (atributos) y comportamientos (métodos). A diferencia de la programación estructurada, la POO modela el mundo real: un Estudiante, una CuentaBancaria o un Vehículo son objetos con características y acciones propias. En Java, el entorno de desarrollo incluye el JDK (Java Development Kit) que provee el compilador javac y herramientas; la JVM (Java Virtual Machine) que ejecuta el bytecode; y los IDEs como IntelliJ IDEA, Eclipse, NetBeans o VS Code que facilitan la escritura, depuración y gestión de proyectos. Los paquetes organizan clases relacionadas y evitan conflictos de nombres.

Ejemplos desarrollados

// MiPrimerPrograma.java - Un programa básico que muestra la estructura de Java
public class MiPrimerPrograma {
    // Método principal: punto de entrada de la aplicación
    public static void main(String[] args) {
        // Imprimimos un mensaje en consola
        System.out.println("¡Hola, POO en Java!");
        System.out.println("JDK: " + System.getProperty("java.version"));
        System.out.println("JVM: " + System.getProperty("java.vm.name"));
    }
}
// Estructura de paquetes
// archivo: com/universidad/poo/Saludo.java
package com.universidad.poo;

public class Saludo {
    private String mensaje;

    public Saludo(String mensaje) {
        this.mensaje = mensaje;
    }

    public void mostrar() {
        System.out.println("Mensaje: " + mensaje);
    }

    public static void main(String[] args) {
        Saludo s = new Saludo("Aprendiendo paquetes en Java");
        s.mostrar();
    }
}

2

Clases y objetos en Java

Creación de clases, objetos y ejemplos prácticos

Explicación conceptual

Una clase es una plantilla o molde que define la estructura y el comportamiento que tendrán los objetos. Piensa en una clase Coche: todos los coches tienen marca, modelo y color (atributos), y todos pueden acelerar o frenar (métodos). Un objeto es una instancia concreta de esa clase: miCoche es un objeto específico de la clase Coche con marca "Toyota", modelo "Corolla" y color "rojo". En Java, los objetos se crean con el operador new, que reserva memoria en el heap e invoca el constructor de la clase.

Ejemplos desarrollados

// Definición de la clase Persona
public class Persona {
    // Atributos
    String nombre;
    int edad;

    // Método para mostrar información
    public void presentarse() {
        System.out.println("Hola, soy " + nombre + " y tengo " + edad + " años.");
    }

    // Método principal para probar la clase
    public static void main(String[] args) {
        // Creación de un objeto (instancia)
        Persona persona1 = new Persona();
        persona1.nombre = "Ana";
        persona1.edad = 22;
        persona1.presentarse();  // Salida: Hola, soy Ana y tengo 22 años.
    }
}
// Clase Libro con múltiples objetos
public class Libro {
    String titulo;
    String autor;
    int anioPublicacion;

    public void mostrarDetalles() {
        System.out.println("'" + titulo + "' por " + autor + " (" + anioPublicacion + ")");
    }

    public static void main(String[] args) {
        // Creamos varios objetos de la misma clase
        Libro libro1 = new Libro();
        libro1.titulo = "Cien Años de Soledad";
        libro1.autor = "Gabriel García Márquez";
        libro1.anioPublicacion = 1967;

        Libro libro2 = new Libro();
        libro2.titulo = "1984";
        libro2.autor = "George Orwell";
        libro2.anioPublicacion = 1949;

        Libro libro3 = new Libro();
        libro3.titulo = "El Principito";
        libro3.autor = "Antoine de Saint-Exupéry";
        libro3.anioPublicacion = 1943;

        libro1.mostrarDetalles();
        libro2.mostrarDetalles();
        libro3.mostrarDetalles();
    }
}

Atributos, métodos y encapsulamiento

Modificadores de acceso, getters y setters

Explicación conceptual

El encapsulamiento es uno de los cuatro pilares de la POO. Consiste en ocultar los detalles internos de una clase y proteger sus datos del acceso directo desde otras clases. Esto se logra declarando los atributos como private y proporcionando getters (métodos para leer) y setters (métodos para modificar) públicos. Los modificadores de acceso en Java son: public (accesible desde cualquier clase), private (solo dentro de la misma clase), protected (mismo paquete y subclases) y default (mismo paquete). El encapsulamiento permite validar datos antes de asignarlos y facilita el mantenimiento del código.

Ejemplos desarrollados

// Clase con encapsulamiento: atributos privados, getters y setters
public class Producto {
    private String nombre;
    private double precio;
    private int stock;

    // Getter para nombre
    public String getNombre() {
        return nombre;
    }

    // Setter para nombre
    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    // Getter para precio
    public double getPrecio() {
        return precio;
    }

    // Setter para precio
    public void setPrecio(double precio) {
        if (precio > 0) {
            this.precio = precio;
        } else {
            System.out.println("El precio debe ser positivo.");
        }
    }

    // Getter y Setter para stock
    public int getStock() { return stock; }
    public void setStock(int stock) {
        if (stock >= 0) this.stock = stock;
    }
}
// Uso del encapsulamiento con validación
public class Tienda {
    public static void main(String[] args) {
        Producto p = new Producto();
        p.setNombre("Laptop HP");
        p.setPrecio(2500.00);
        p.setStock(10);

        System.out.println("Producto: " + p.getNombre());
        System.out.println("Precio: $" + p.getPrecio());
        System.out.println("Stock: " + p.getStock());

        // Intento de asignar precio inválido
        p.setPrecio(-100);  // Mensaje: El precio debe ser positivo.
        System.out.println("Precio tras intento inválido: $" + p.getPrecio()); // Sigue 2500
    }
}

Constructores y representación de objetos

Constructores, método toString() y sobrecarga

Explicación conceptual

Un constructor es un método especial que se invoca automáticamente al crear un objeto con new. Su nombre coincide con el de la clase y no tiene tipo de retorno. Sirve para inicializar los atributos del objeto en un estado válido. Java proporciona un constructor por defecto (sin parámetros) si no se define ninguno. La sobrecarga permite tener varios constructores con diferentes parámetros. El método toString() heredado de Object devuelve una representación en texto del objeto; se recomienda sobrescribirlo para mostrar información significativa en lugar del hash code.

Ejemplos desarrollados

// Clase con constructores sobrecargados
public class Rectangulo {
    private double ancho;
    private double alto;

    // Constructor sin parámetros (valores por defecto)
    public Rectangulo() {
        this.ancho = 1.0;
        this.alto = 1.0;
    }

    // Constructor con parámetros
    public Rectangulo(double ancho, double alto) {
        this.ancho = ancho;
        this.alto = alto;
    }

    // Constructor para un cuadrado (un solo parámetro)
    public Rectangulo(double lado) {
        this.ancho = lado;
        this.alto = lado;
    }

    public double calcularArea() {
        return ancho * alto;
    }
}
// Sobrescritura de toString() para representación clara
public class Estudiante {
    private String nombre;
    private int codigo;
    private double nota;

    public Estudiante(String nombre, int codigo, double nota) {
        this.nombre = nombre;
        this.codigo = codigo;
        this.nota = nota;
    }

    @Override
    public String toString() {
        return "Estudiante{" +
               "nombre='" + nombre + '\'' +
               ", codigo=" + codigo +
               ", nota=" + nota +
               '}';
    }

    public static void main(String[] args) {
        Estudiante e = new Estudiante("Carlos", 20241001, 4.5);
        System.out.println(e); // Llama automáticamente a toString()
        // Salida: Estudiante{nombre='Carlos', codigo=20241001, nota=4.5}
    }
}

Herencia en Java

Herencia simple y uso de super()

Explicación conceptual

La herencia es un mecanismo que permite crear una clase nueva (subclase o clase hija) a partir de una clase existente (superclase o clase padre). La subclase hereda todos los atributos y métodos de la superclase y puede agregar nuevos o modificar los existentes. En Java, la herencia es simple: una clase solo puede tener una superclase directa (se usa extends). La palabra clave super permite acceder al constructor y a los miembros de la clase padre. La herencia promueve la reutilización de código y establece relaciones jerárquicas naturales (un Perro es un Animal).

Ejemplos desarrollados

// Superclase
public class Animal {
    protected String nombre;
    protected int edad;

    public Animal(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    public void hacerSonido() {
        System.out.println("El animal hace un sonido.");
    }
}

// Subclase
public class Perro extends Animal {
    private String raza;

    public Perro(String nombre, int edad, String raza) {
        super(nombre, edad);  // Llama al constructor de Animal
        this.raza = raza;
    }

    @Override
    public void hacerSonido() {
        System.out.println(nombre + " dice: ¡Guau guau!");
    }

    public void mostrarInfo() {
        System.out.println("Perro: " + nombre + ", Edad: " + edad + ", Raza: " + raza);
    }
}
// Demostración de herencia con super()
public class Zoologico {
    public static void main(String[] args) {
        Perro miPerro = new Perro("Rex", 3, "Labrador");
        miPerro.mostrarInfo();    // Perro: Rex, Edad: 3, Raza: Labrador
        miPerro.hacerSonido();    // Rex dice: ¡Guau guau!

        // También podemos tratar al Perro como Animal (polimorfismo)
        Animal animalRef = miPerro;
        animalRef.hacerSonido();  // Rex dice: ¡Guau guau! (método sobrescrito)
    }
}

Polimorfismo y sobreescritura de métodos

Ejercicios prácticos de polimorfismo dinámico

Explicación conceptual

El polimorfismo (del griego "muchas formas") permite que una misma referencia de tipo superclase pueda referirse a objetos de diferentes subclases, y que el método invocado sea el correspondiente al objeto real en tiempo de ejecución (polimorfismo dinámico). Esto se logra mediante la sobrescritura (@Override) de métodos. Por ejemplo, una variable de tipo Animal puede referenciar un Perro o un Gato; al llamar hacerSonido(), se ejecutará la versión específica de cada subclase. El polimorfismo hace el código más extensible y mantenible, pues se pueden agregar nuevas subclases sin modificar el código existente.

Ejemplos desarrollados

// Polimorfismo con animales
public class DemoPolimorfismo {
    public static void main(String[] args) {
        // Referencia de tipo Animal, objetos de subclases
        Animal[] zoo = new Animal[3];
        zoo[0] = new Perro("Rex", 3, "Labrador");
        zoo[1] = new Gato("Misu", 2, "Siamés");
        zoo[2] = new Perro("Luna", 1, "Bulldog");

        // Polimorfismo: cada objeto ejecuta su propio hacerSonido()
        for (Animal a : zoo) {
            a.hacerSonido();
        }
        // Salida:
        // Rex dice: ¡Guau guau!
        // Misu dice: ¡Miau miau!
        // Luna dice: ¡Guau guau!
    }
}
// Polimorfismo con figuras geométricas
public abstract class Figura {
    public abstract double calcularArea();
}

public class Circulo extends Figura {
    private double radio;
    public Circulo(double r) { this.radio = r; }
    @Override
    public double calcularArea() {
        return Math.PI * radio * radio;
    }
}

public class Triangulo extends Figura {
    private double base, altura;
    public Triangulo(double b, double h) { base = b; altura = h; }
    @Override
    public double calcularArea() {
        return (base * altura) / 2;
    }
}

// Uso polimórfico
public class CalculadoraAreas {
    public static void main(String[] args) {
        Figura[] figuras = {
            new Circulo(5),
            new Triangulo(4, 3),
            new Circulo(2.5)
        };
        for (Figura f : figuras) {
            System.out.printf("Área: %.2f%n", f.calcularArea());
        }
    }
}

Clases abstractas e interfaces

Implementación con clases abstractas e interfaces

Explicación conceptual

Una clase abstracta (abstract class) es una clase que no se puede instanciar directamente; sirve como plantilla para otras clases. Puede contener métodos abstractos (sin implementación, que las subclases deben definir) y métodos concretos (con implementación). Una interfaz (interface) define un contrato de comportamiento: solo declara métodos (sin cuerpo) y constantes, y una clase puede implementar múltiples interfaces. La diferencia clave es que una clase solo puede extender una clase abstracta, pero puede implementar varias interfaces, lo que da mayor flexibilidad. Las interfaces son ideales para definir capacidades transversales como Comparable o Serializable.

Ejemplos desarrollados

// Clase abstracta VehiculoAcuatico
public abstract class VehiculoAcuatico {
    protected String nombre;
    protected double eslora; // longitud en metros

    public VehiculoAcuatico(String nombre, double eslora) {
        this.nombre = nombre;
        this.eslora = eslora;
    }

    // Método abstracto: cada subclase debe implementarlo
    public abstract String tipoPropulsion();

    // Método concreto: compartido por todas las subclases
    public void zarpar() {
        System.out.println(nombre + " está zarpando del puerto.");
    }
}

public class Velero extends VehiculoAcuatico {
    private int numVelas;

    public Velero(String nombre, double eslora, int numVelas) {
        super(nombre, eslora);
        this.numVelas = numVelas;
    }

    @Override
    public String tipoPropulsion() {
        return "Propulsión a vela con " + numVelas + " velas.";
    }
}
// Interfaces: definen capacidades
public interface Volador {
    void volar();
}

public interface Nadador {
    void nadar();
}

// Una clase puede implementar múltiples interfaces
public class Pato implements Volador, Nadador {
    private String nombre;

    public Pato(String nombre) { this.nombre = nombre; }

    @Override
    public void volar() {
        System.out.println(nombre + " está volando hacia el sur.");
    }

    @Override
    public void nadar() {
        System.out.println(nombre + " está nadando en el lago.");
    }
}

public class Avion implements Volador {
    private String modelo;
    public Avion(String m) { this.modelo = m; }

    @Override
    public void volar() {
        System.out.println("Avión " + modelo + " despegando...");
    }
}

Composición y modularización del código

Relación de objetos y diseño orientado a componentes

Explicación conceptual

La composición es un principio de diseño donde una clase contiene instancias de otras clases como atributos, estableciendo una relación "tiene un" (has-a) en lugar de "es un" (is-a) de la herencia. Por ejemplo, un Coche tiene un Motor, no es un Motor. La composición favorece la modularización: cada componente tiene una responsabilidad clara y puede desarrollarse, probarse y mantenerse de forma independiente. Este enfoque sigue el principio de "preferir composición sobre herencia" y facilita la reutilización y la flexibilidad del código.

Ejemplos desarrollados

// Composición: Coche "tiene un" Motor y "tiene" Ruedas
public class Motor {
    private int cilindrada;
    private String tipo;

    public Motor(int cilindrada, String tipo) {
        this.cilindrada = cilindrada;
        this.tipo = tipo;
    }

    public void encender() {
        System.out.println("Motor " + tipo + " (" + cilindrada + "cc) encendido.");
    }
}

public class Rueda {
    private String marca;
    private int diametro;

    public Rueda(String marca, int diametro) {
        this.marca = marca;
        this.diametro = diametro;
    }
}

public class Coche {
    private String modelo;
    private Motor motor;        // Composición
    private Rueda[] ruedas;     // Composición (arreglo de 4 ruedas)

    public Coche(String modelo, Motor motor) {
        this.modelo = modelo;
        this.motor = motor;
        this.ruedas = new Rueda[4];
        for (int i = 0; i < 4; i++) {
            ruedas[i] = new Rueda("Michelin", 17);
        }
    }
}
// Diseño modular: separación por responsabilidades
// Módulo de autenticación
public class Autenticador {
    public boolean login(String usuario, String clave) {
        // Lógica de autenticación
        return usuario.equals("admin") && clave.equals("1234");
    }
}

// Módulo de registro de actividad
public class Logger {
    public void registrar(String mensaje) {
        System.out.println("[LOG] " + mensaje);
    }
}

// Clase principal que compone los módulos
public class Aplicacion {
    private Autenticador auth;
    private Logger logger;

    public Aplicacion() {
        this.auth = new Autenticador();
        this.logger = new Logger();
    }

    public void iniciar(String usuario, String clave) {
        logger.registrar("Intento de inicio de sesión: " + usuario);
        if (auth.login(usuario, clave)) {
            logger.registrar("Inicio exitoso.");
        } else {
            logger.registrar("Fallo de autenticación.");
        }
    }
}

Manejo de errores y excepciones

try, catch, throw, throws y finally

Explicación conceptual

Las excepciones son eventos que interrumpen el flujo normal de un programa (errores en tiempo de ejecución). Java maneja las excepciones con un mecanismo de bloques: try (código que puede fallar), catch (captura y maneja la excepción), finally (se ejecuta siempre, haya o no excepción, ideal para liberar recursos). Las excepciones pueden ser checked (se declaran con throws y el compilador obliga a manejarlas) o unchecked (subclases de RuntimeException, no obligatorias). La palabra clave throw permite lanzar una excepción manualmente, útil para validaciones personalizadas.

Ejemplos desarrollados

// Manejo de excepciones con try-catch-finally
import java.util.Scanner;

public class DivisionSegura {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        try {
            System.out.print("Numerador: ");
            int a = sc.nextInt();
            System.out.print("Denominador: ");
            int b = sc.nextInt();
            int resultado = a / b;
            System.out.println("Resultado: " + resultado);
        } catch (ArithmeticException e) {
            System.out.println("Error: No se puede dividir por cero.");
        } catch (Exception e) {
            System.out.println("Error inesperado: " + e.getMessage());
        } finally {
            sc.close();
            System.out.println("Scanner cerrado correctamente.");
        }
    }
}
// Lanzamiento y propagación de excepciones
public class CuentaBancaria {
    private double saldo;

    public CuentaBancaria(double saldoInicial) {
        this.saldo = saldoInicial;
    }

    // Declara que puede lanzar una excepción
    public void retirar(double monto) throws SaldoInsuficienteException {
        if (monto > saldo) {
            throw new SaldoInsuficienteException(
                "Saldo insuficiente. Disponible: $" + saldo
            );
        }
        saldo -= monto;
        System.out.println("Retiro exitoso. Nuevo saldo: $" + saldo);
    }
}

// Excepción personalizada
class SaldoInsuficienteException extends Exception {
    public SaldoInsuficienteException(String mensaje) {
        super(mensaje);
    }
}

Colecciones y estructuras de datos

List, Map, Set y sus implementaciones

Explicación conceptual

El Java Collections Framework proporciona estructuras de datos reutilizables y eficientes. Las principales interfaces son: List (colección ordenada que permite duplicados — ej. ArrayList, LinkedList), Set (colección sin duplicados — ej. HashSet, TreeSet) y Map (pares clave-valor, donde las claves son únicas — ej. HashMap, TreeMap). Cada implementación tiene características distintas de rendimiento: ArrayList es rápido para acceso aleatorio (índice), LinkedList para inserciones/eliminaciones frecuentes; HashSet ofrece búsqueda en O(1), TreeSet mantiene los elementos ordenados.

Ejemplos desarrollados

import java.util.*;

public class DemoColecciones {
    public static void main(String[] args) {
        // List: permite duplicados, mantiene orden de inserción
        List frutas = new ArrayList<>();
        frutas.add("Manzana");
        frutas.add("Banana");
        frutas.add("Manzana");  // Duplicado permitido
        frutas.add("Naranja");
        System.out.println("List: " + frutas); // [Manzana, Banana, Manzana, Naranja]

        // Set: no permite duplicados
        Set frutasUnicas = new HashSet<>();
        frutasUnicas.add("Manzana");
        frutasUnicas.add("Banana");
        frutasUnicas.add("Manzana");  // No se agrega (duplicado)
        System.out.println("Set: " + frutasUnicas); // [Banana, Manzana] (orden no garantizado)

        // TreeSet: ordenado
        Set frutasOrdenadas = new TreeSet<>(frutasUnicas);
        System.out.println("TreeSet: " + frutasOrdenadas); // [Banana, Manzana]
    }
}
import java.util.*;

public class DirectorioTelefonico {
    public static void main(String[] args) {
        // Map: pares clave-valor
        Map directorio = new HashMap<>();
        directorio.put("Ana", "555-0101");
        directorio.put("Carlos", "555-0102");
        directorio.put("Beatriz", "555-0103");
        directorio.put("Ana", "555-0199"); // Sobrescribe el valor anterior

        // Recorrer el Map
        for (Map.Entry entrada : directorio.entrySet()) {
            System.out.println(entrada.getKey() + " -> " + entrada.getValue());

        }
    }
}

Lectura y escritura de archivos en Java

Trabajo con archivos .txt y .csv

Explicación conceptual

Java ofrece múltiples formas de leer y escribir archivos. Para texto simple se usan clases como FileReader/FileWriter (caracteres) o BufferedReader/BufferedWriter (lectura eficiente línea por línea). Para archivos CSV, es común usar BufferedReader con split() o la clase Scanner. Desde Java 7, se recomienda usar try-with-resources que cierra automáticamente los recursos al finalizar el bloque. También existen FileInputStream/FileOutputStream para datos binarios. Siempre debe manejarse IOException para garantizar robustez.

Ejemplos desarrollados

// Lectura y escritura de archivos .txt con try-with-resources
import java.io.*;

public class ManejadorArchivos {
    public static void escribirArchivo(String ruta, String contenido) {
        // try-with-resources cierra automáticamente el BufferedWriter
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(ruta))) {
            writer.write(contenido);
            System.out.println("Archivo escrito exitosamente.");
        } catch (IOException e) {
            System.err.println("Error al escribir: " + e.getMessage());
        }
    }

    public static void leerArchivo(String ruta) {
        try (BufferedReader reader = new BufferedReader(new FileReader(ruta))) {
            String linea;
            int numLinea = 1;
            while ((linea = reader.readLine()) != null) {
                System.out.println(numLinea++ + ": " + linea);
            }
        } catch (FileNotFoundException e) {
            System.err.println("Archivo no encontrado: " + ruta);
        } catch (IOException e) {
            System.err.println("Error al leer: " + e.getMessage());
        }
    }
}
// Procesamiento de archivo CSV
import java.io.*;
import java.util.*;

public class ProcesadorCSV {
    public static List leerCSV(String ruta) {
        List registros = new ArrayList<>();

        try (BufferedReader reader = new BufferedReader(new FileReader(ruta))) {
            String linea;
            // Leer encabezados (opcional)
            String encabezados = reader.readLine();
            System.out.println("Encabezados: " + encabezados);

            while ((linea = reader.readLine()) != null) {
                // split con coma, acepta comillas para campos con coma interna
                String[] campos = linea.split(",(?=(?:[^\"]*\"[^\"]*\")*[^\"]*$)");
                registros.add(campos);
            }
        } catch (IOException e) {
            System.err.println("Error procesando CSV: " + e.getMessage());
        }
        return registros;
    }

    public static void main(String[] args) {
        List datos = leerCSV("estudiantes.csv");
        for (String[] fila : datos) {
            System.out.println("Nombre: " + fila[0] + ", Nota: " + fila[1]);
        }
    }
}

Buenas prácticas de programación en Java y refactorización

Revisión de código y herramientas como Checkstyle y SonarLint

Explicación conceptual

Las buenas prácticas en Java incluyen: usar nombres descriptivos en inglés o español consistente, seguir convenciones de código (camelCase para métodos, PascalCase para clases), mantener métodos cortos con una sola responsabilidad, evitar números mágicos usando constantes, y escribir comentarios que expliquen el "por qué" no el "qué". La refactorización es el proceso de mejorar el diseño del código sin cambiar su comportamiento externo. Herramientas como Checkstyle (verifica convenciones de estilo), SonarLint (detecta bugs y code smells en tiempo real) y PMD (análisis estático) ayudan a mantener la calidad del código automáticamente.

Ejemplos desarrollados

// CÓDIGO SIN BUENAS PRÁCTICAS (Antes)
public class calc {
    public double f(double a, double b, int o) {
        if(o == 1) { return a + b; }
        else if(o == 2) { return a - b; }
        else if(o == 3) { return a * b; }
        else if(o == 4) {
            if(b != 0) return a / b;
            else return 0;
        }
        return 0;
    }
}

// CÓDIGO REFACTORIZADO (Después) - Nombres claros, sin números mágicos
public class Calculadora {
    private static final int OPERACION_SUMA = 1;
    private static final int OPERACION_RESTA = 2;
    private static final int OPERACION_MULTIPLICACION = 3;
    private static final int OPERACION_DIVISION = 4;

    public double calcular(double operando1, double operando2, int tipoOperacion) {
        switch (tipoOperacion) {
            case OPERACION_SUMA: return sumar(operando1, operando2);
            case OPERACION_RESTA: return restar(operando1, operando2);
            case OPERACION_MULTIPLICACION: return multiplicar(operando1, operando2);
            case OPERACION_DIVISION: return dividir(operando1, operando2);
            default: throw new IllegalArgumentException("Operación no válida: " + tipoOperacion);
        }
    }

    private double sumar(double a, double b) { return a + b; }
    private double restar(double a, double b) { return a - b; }
    private double multiplicar(double a, double b) { return a * b; }
    private double dividir(double a, double b) {
        if (b == 0) throw new ArithmeticException("División por cero no permitida.");
        return a / b;
    }
}
// Refactorización: extraer método y eliminar duplicación
public class ProcesadorPedidos {
    private static final double IMPUESTO = 0.19; // Constante en lugar de número mágico
    private static final double DESCUENTO_MINIMO = 1000.0;

    public double calcularTotal(double subtotal) {
        double impuesto = subtotal * IMPUESTO;
        double descuento = calcularDescuento(subtotal);
        return subtotal + impuesto - descuento;
    }

    // Método extraído: antes estaba duplicado en varias partes
    private double calcularDescuento(double subtotal) {
        return subtotal > DESCUENTO_MINIMO ? subtotal * 0.10 : 0.0;
    }
}
